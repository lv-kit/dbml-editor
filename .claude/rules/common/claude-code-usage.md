# claude-code-usage（トークン最適化の正本）

> **正本**: モデル選択・subagent 委譲・context 管理の方針。

## 1. モデルルーティング（3段）

| 段 | モデル | 用途 |
|---|---|---|
| 実装・検査 | `claude-sonnet-5` / `medium` | 通常の実装・テスト修正・ビルド修正・ドキュメント同期 |
| 設計・レビュー | `claude-opus-4-8` / `high` | 設計・計画・汎用/専門セルフレビュー |
| 調査 | `claude-sonnet-5` / `medium` | コード探索・事実確認 |

- 高推論モデルは**計画/設計/セルフレビューに限定**。単純作業に使わない

## 2. subagent 委譲

- 大量の tool call を伴う調査は `code-explorer` / `Explore` に委譲し、結果のダイジェストだけ受け取る（main context を汚さない）
- レビューは `code-reviewer` と変更領域に応じた専門 reviewer へ。指摘がゼロになるまで修正・影響検査・同じ観点の再レビューを繰り返す
- agent に指定したモデルと effort は無断で別モデルへ置換しない。利用不可の場合は停止して不足を報告する
- agy（Antigravity/Gemini）への委譲は break-even を超える一括作業のみ（大量スキャフォールド・網羅テスト生成・移行）。小さいタスクは自分でやる

## 3. context 管理

- 検索は semble 第一選択（`.claude/rules/common/code-search.md`）。同じ内容を再検索しない
- 一度読んだファイルを検証目的で読み直さない
- 関連のないタスクは `/clear` で分ける。圧縮時は変更ファイルと検証コマンドを要約に残す
- 長い調査結果は貼り戻さず、要点だけを残す
