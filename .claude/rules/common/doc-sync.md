# doc-sync（ドキュメント同期の正本）

> **正本**: コードとセットで更新すべきドキュメントと、その同期規則。どの AI ツールでも適用する。

## 原則

- **AI との会話で仕様が固まったら、必ず仕様ドキュメントに記載する**（実装と同一コミット）
- コードとドキュメントを別コミットに分けない。ドリフトを作らない

## 同期対象

| 変更 | 同時に更新するドキュメント |
|---|---|
| DB スキーマ（`web/src/lib/server/db/schema.ts`） | スキーマ由来のドキュメント（新設時は `docs/schema.md` 等）、Drizzle マイグレーション |
| 環境変数の追加/変更 | `web/.env.example` |
| 公開 API・ルートの追加/変更 | 該当ルートの README / 仕様メモ |
| ハーネス構成（hooks/agents/commands 数） | `.claude/README.md`（`/sync-claude-md` で検出） |

> 現状 `docs/` ディレクトリは未整備。仕様が増えたら `docs/` を新設し、本表に追記する。

## ドリフト防止

- 大規模リファクタ・依存更新後は `/sync-claude-md` で実装とドキュメントの乖離を検出する
