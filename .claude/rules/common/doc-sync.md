# doc-sync（ドキュメント同期の正本）

> **正本**: コードとセットで更新すべきドキュメントと、その同期規則。どの AI ツールでも適用する。

## 原則

- **AI との会話で仕様が固まったら、必ず仕様ドキュメントに記載する**（実装と同一コミット）。対象はインタビューの回答、質問への回答、レビューでの合意、実装中に確定した仕様判断を含む
- 決定事項は確定直後、遅くとも対応する実装の完了前に反映する。複数回の会話で決まった内容を最後にまとめて推測で補完してはならない
- 反映先となる仕様書がなければ、対象機能を説明する `docs/` 配下の Markdown を新設する。タスクキューや引き継ぎノートは仕様書の代わりにしない
- コードとドキュメントを別コミットに分けない。ドリフトを作らない

## 仕様書に記録する内容

- 利用者向けの振る舞い、受け入れ条件、例外・失敗時の扱い
- 入出力、データ制約、権限、互換性に関する決定
- 複数案から選んだ場合の結論と、その判断に必要な理由

会話の逐語録や個人情報は記録しない。実装に必要な、検証可能な決定事項だけを簡潔に記録する。

## 同期対象

| 変更 | 同時に更新するドキュメント |
|---|---|
| DB スキーマ（`web/src/lib/server/db/schema.ts`） | スキーマ由来のドキュメント（新設時は `docs/schema.md` 等）、Drizzle マイグレーション |
| 環境変数の追加/変更 | `web/.env.example` |
| 公開 API・ルートの追加/変更 | 該当ルートの README / 仕様書 |
| インタビュー・会話で確定した機能要件 | 該当機能の仕様書（なければ `docs/` に新設） |
| ハーネス構成（hooks/agents/commands 数） | `.claude/README.md`（`/sync-claude-md` で検出） |

> 現状 `docs/` ディレクトリは未整備。会話で確定した要件または仕様が増えた時点で、該当仕様書を `docs/` に新設する。

## ドリフト防止

- 大規模リファクタ・依存更新後は `/sync-claude-md` で実装とドキュメントの乖離を検出する
