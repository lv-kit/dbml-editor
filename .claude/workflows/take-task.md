# workflow: take-task

`/take-task` の詳細手順。管理セッションが用意したタスクを実装セッションが受け取る。

## 手順
1. 対象タスクを決める
   - 引数 `TASK-番号` があればそれ
   - 無指定なら `.codex/tasks/backlog/` の先頭（番号昇順）
2. タスクファイルを Read し、以下を把握する
   - 前提状態 / 実施内容 / 触ってよい範囲 / 触ってはいけない範囲 / 制約 / テストコマンド / 完了条件
3. ファイルを移動する
   ```bash
   git mv .codex/tasks/backlog/TASK-XXX-*.md .codex/tasks/in-progress/
   ```
4. ブランチを確認する。`main` の場合は候補名・派生元・理由を人間へ提示し、確認前に作成・切替しない。既存の適切な作業ブランチ指定があればそのまま継続する
5. `/plan` へ進む

## 注意
- タスクに曖昧さ・不足があれば、実装前にユーザーへ確認する（推測で進めない）
- 「触ってはいけない範囲」を厳守する
